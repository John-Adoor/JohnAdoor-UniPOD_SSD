Week 2: My Digital Workshop Notebook

Date: 1st October to 6 October 2025

Quick Summary

  • Skills Learned: Git version control, GitHub repository management, SSH key setup, VS Code usage
  • Digital Tools and Software Used: Git, Git Bash, VS Code, GitHub, SSH keys
  • Topics: Documentation, Version Control, Repository Management
  • Key Concepts: Git workflow (add, commit, push), SSH authentication, GitHub Pages deployment
  • Challenges Faced: Setting up SSH keys, understanding Git workflow, deploying with GitHub Pages
 

Introduction

Week 2 focused on establishing a digital workshop notebook using GitHub as the primary documentation platform. The week emphasized the importance of documentation as a "superpower" for personal reference, career development, collaboration, and program submission. As the Chinese proverb in the materials states: "The palest ink is better than the best memory."

The core objective was to master the tools and workflows for effective project documentation, including Git for version control, VS Code as a text editor, and GitHub for hosting repositories. The week culminated in creating a personal portfolio repository and deploying it as a live website using GitHub Pages.

Process Documentation

Step 1: Tool Installation & Setup

Installed essential development tools:

  • Git & Git Bash: Downloaded from git-scm.com with default settings
  • VS Code: Installed as the primary code editor from code.visualstudio.com
  • GitHub Account: Created with professional username

Figure 1: Git and VS Code installation process

Step 2: SSH Key Setup

Configured secure password-less connection to GitHub:

  • Generated SSH key pair using: ssh-keygen -t ed25519 -C "your_email@example.com"
  • Copied public key using cat ~/.ssh/id_ed25519.pub
  • Added public key to GitHub Settings under SSH and GPG keys

Figure 2: SSH key generation and GitHub configuration

Step 3: Repository Creation

Created and configured GitHub repository:

  • Named using format: username-UniPod_Name
  • Set as Public visibility
  • Initialized with README file
  • Cloned to local machine: git clone git@github.com:username/repo-name.git

Figure 3: Creating and cloning repository on GitHub

Step 4: Git Workflow Implementation

Practiced the essential three-step Git workflow:

  1. Edited README.md in VS Code
  2. Staged changes with git add README.md
  3. Committed with descriptive message: git commit -m "Update README"
  4. Pushed changes: git push origin main

Figure 4: Git add, commit, and push workflow in terminal

Step 5: GitHub Pages Deployment

Deployed repository as live website:

  • Enabled GitHub Pages in repository Settings
  • Selected "Deploy from a branch" as source
  • Chose main branch and root folder
  • Site published at: https://username.github.io/repository-name/

Figure 5: GitHub Pages configuration and live website

Reflection

This week provided foundational skills in documentation and version control that are essential for modern software development and digital fabrication workflows. The process of setting up SSH keys initially seemed complex, but the analogy of a "lock and key" helped conceptualize the security mechanism. Understanding the three-step Git workflow (add, commit, push) was crucial - it transforms from being just commands to a logical process of preparing, documenting, and sharing changes.

The most valuable insight was realizing how documentation serves multiple purposes: as a personal memory aid, a professional portfolio, and a collaboration tool. Creating a live website through GitHub Pages demonstrated how technical documentation can become immediately accessible and shareable. This aligns with the program's emphasis on building a digital presence and portfolio.

Moving forward, maintaining consistent documentation habits will be essential. The skills learned this week establish the groundwork for all future projects in the program, ensuring that work is properly versioned, documented, and shareable. The emphasis on documentation as a "new professional habit" resonates with the long-term goal of building a comprehensive portfolio that demonstrates growth and capability throughout the UniPod program.
